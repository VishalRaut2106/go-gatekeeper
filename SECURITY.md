# Security Policy

We take the security of Gatekeeper Shell seriously. Because this application facilitates remote terminal access, ensuring connection integrity and data privacy is our top priority.

## Reporting a Vulnerability

If you discover a security vulnerability, **please do not open a public GitHub issue.** Doing so publicizes the flaw before we have a chance to patch it, potentially exposing active users to risk.

Instead, please report vulnerabilities by contacting us directly:
* Email: **rautv2005@gmail.com**

Please include as much detail as possible, including:
- A description of the vulnerability and its potential impact.
- Steps to reproduce the issue (and any proof-of-concept code, if available).
- The platform details (CLI client OS, server details).

## Our Response Process

1. **Acknowledgment:** We will acknowledge receipt of your vulnerability report within 48 hours.
2. **Investigation:** We will investigate the issue and coordinate with you to understand the scope and root cause.
3. **Remediation:** We aim to release a patch or mitigation guidelines as quickly as possible.
4. **Disclosure:** Once the patch is deployed and stable, we will publicly disclose the vulnerability with credit to you for discovering it responsibly.
